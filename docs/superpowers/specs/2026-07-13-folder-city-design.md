# 文件夹城市（Folder City）MVP 设计规格

## 目标与边界

构建一个本地运行的浏览器应用：用户选择一个文件夹后，应用只在浏览器内读取该文件夹及子目录中文件的元数据，并把它可视化为可探索的二维城市。

本期目标是完成一个可分享、可交互且稳定的 MVP。项目不读取文件内容、不上传任何文件或元数据、不接入 AI API，也不提供云端账号、持久化存储或多人协作。

## 实现方案比较

1. **推荐：React + TypeScript + Vite，二维 SVG 城市。** SVG 的每栋建筑都是可访问的 DOM 元素，便于点击、提示、筛选和小规模动画；项目无外部后端，适合在 3–5 小时完成。
2. **Canvas 二维城市。** 绘制性能更高，但命中测试、键盘访问和建筑详情交互需要额外实现。本 MVP 不需要处理超大目录，收益不足。
3. **Three.js 三维城市。** 视觉上限高，但相机控制、性能调优和移动端适配会明显扩大范围，不符合本期时间约束。

采用方案 1。默认设定最多展示 1,000 个文件；超出时按最近修改时间保留 1,000 个文件，并明确显示截断提示。目录本身不作为建筑。

## 用户流程与数据流

1. 用户打开应用，看到用途说明、隐私说明和“选择文件夹”按钮。
2. 浏览器通过 File System Access API 请求文件夹访问权限；不支持该 API 的浏览器显示明确提示，并提供不可用状态，不实现 ZIP 上传替代方案。
3. 应用递归遍历文件夹，收集文件的名称、相对路径、大小、扩展名/推断类型、最后修改时间；不读取文件内容。
4. 应用对数据排序、截断，并转换为城市建筑模型；城市视图、统计栏和筛选器均消费同一内存数据集。
5. 用户缩放、平移、筛选或点击建筑；点击后在侧栏显示该文件的元数据和由规则模板生成的“文件故事”。

文件数据仅存在于当前浏览器页面内存；刷新页面即丢失。

## 城市映射规则

| 源数据 | 可视化规则 |
| --- | --- |
| 文件大小 | 建筑高度。使用对数缩放，避免少数大文件压扁其他建筑；高度限制在统一最小/最大值内。 |
| 文件类型 | 街区与颜色。文档、图片、视频音频、代码/配置、压缩包、其他六类；无扩展名归入“其他”。 |
| 最后修改时间 | 建筑外观新旧。近 7 天为明亮新建筑，8–90 天为正常，超过 90 天为低饱和旧建筑。 |
| 相对路径 | 街区内的建筑分组与标签；目录深度影响排列分区，不影响高度。 |
| 文件名与属性 | 点击侧栏中的模板故事，例如“这座建筑属于图片街区，最近一次翻新于 2026-07-01，占地 2.4 MB”。 |

建筑布局必须稳定：相同输入和排序规则应产生相同位置。文件按“相对路径”排序后依次排列在所属街区网格中。

## 应用结构

- `fileSystem`：权限请求、目录递归、元数据提取、数量限制和异常归一化。
- `cityModel`：文件类型分类、时间分级、大小高度计算、稳定布局与故事文本生成；保持为纯函数，便于测试。
- `CityCanvas`：SVG 城市、平移缩放、悬停和建筑选中。
- `Controls`：文件夹选择、文件类型筛选、时间筛选与重置。
- `DetailsPanel`：展示选中文件的信息和文件故事。
- `StatsBar`：文件数量、总大小、最大文件、类型分布摘要和截断状态。

组件之间仅传递明确的模型与回调；文件系统访问逻辑不进入 UI 组件，城市规则不依赖浏览器 API。

## 交互与状态

- 初始状态：说明与选择文件夹按钮。
- 扫描状态：显示扫描进度和取消提示；扫描异常后显示可理解的错误信息。
- 成功状态：显示城市、统计和筛选控件。
- 空目录：显示空城市及重新选择按钮。
- 选中状态：建筑带可见高亮，详情侧栏打开；点击空白处或关闭按钮取消选中。
- 筛选状态：隐藏不匹配的建筑并更新统计；不重新扫描文件夹。

平移与缩放仅改变视图变换，不改动建筑模型。鼠标与触控板均应可操作；建筑可通过键盘聚焦并用 Enter 打开详情。

## 错误处理与兼容性

- 未支持 File System Access API：说明需要使用支持该 API 的 Chromium 系浏览器。
- 用户拒绝权限：保留初始界面并说明未读取任何数据。
- 扫描中遇到不可访问的条目：跳过该条目、计数并在结果中提示。
- 超过上限：显示“仅展示最近修改的 1,000 个文件”的提示，避免声称已呈现完整目录。
- 目录大于 MVP 能力时：不承诺实时渲染或后台扫描；优先保证页面可用和错误透明。

## 视觉与可访问性

城市采用深色背景配高对比霓虹街区色，保持“个人数字城市”的视觉冲击。信息不只依赖颜色：类型用图例和文字标记，新旧程度用亮度与纹理共同表达。

控件和建筑必须具备可见焦点、文本标签与键盘操作。详情信息使用可复制文本；不使用仅悬停可见的关键内容。

## 验证标准

1. TypeScript 类型检查与生产构建均通过。
2. 单元测试覆盖分类、对数高度、时间分级、稳定布局和故事生成。
3. 手动验证：空目录、混合类型目录、权限拒绝、无 API 支持提示、筛选、缩放平移、键盘选择，以及超过 1,000 个文件的截断提示。
4. 通过浏览器检查确认：应用不会发起网络请求来上传文件数据。

## 明确不在 MVP 范围内

- 真实 3D、WebGL、自动生成视频或图片。
- 文件内容预览、文件编辑、删除或任何磁盘写入。
- ZIP 上传、拖入本地目录、Safari/Firefox 兼容性替代实现。
- 城市导出、分享链接、登录、云端同步或 AI 叙事。
