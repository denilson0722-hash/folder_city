# 文件夹城市（Folder City）

一个只在本地浏览器内运行的文件夹元数据可视化 MVP。选择文件夹后，应用将文件类型、大小和最后修改时间映射为可缩放、可平移、可筛选的二维 SVG 城市。

## 浏览层级与显示策略

- **全城级**：浏览全部街区；选择街区可进入街区级。
- **街区级**：查看该街区的每个文件，并可返回全城。
- **建筑级**：选择一栋建筑后，在信息面板查看该文件的精确相对路径与故事；筛选仍以原始文件模型为准。

地图会根据容器尺寸自动适应全部街区、标题和建筑边界，并保留 6% 安全边距。可使用鼠标滚轮、拖动和地图控制按钮缩放、平移或重新适应视图。

全城级根据源文件数量控制渲染量，但统计始终保持精确：

- 1–180 个文件：逐栋显示；
- 181–600 个文件：每个街区最多显示 24 个代表建筑，并用数量徽章汇总其余文件；
- 601–1,000 个文件：按文件类型、一级目录和新旧程度聚合。

进入街区后仍使用精确文件列表；超大街区只绘制当前视野及 12% 预加载范围内的建筑，不会改变源文件总数。

## 运行

```bash
npm install
npm run dev
```

开发服务器启动后，在浏览器中打开终端显示的本地地址。

## 验证与构建

```bash
npm test
npm run typecheck
npm run build
```

生产构建结果位于 `dist/`。

## 浏览器要求

选择文件夹依赖 File System Access API，因此需要支持该 API 的 Chromium 系浏览器（例如最新 Chrome 或 Edge）。不支持时，应用会说明该限制，且不会提供 ZIP 上传等替代入口。

## 隐私边界

- 只读取文件名、相对路径、大小、类型和最后修改时间等元数据。
- 不读取文件内容，不会调用文件内容读取 API。
- 不上传或保存任何文件内容或元数据；没有后端、账号、云同步或浏览器持久化。
- 数据仅保留在当前页面内存中，刷新页面即丢失。
